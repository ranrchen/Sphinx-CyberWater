ForcingDataFileGenerator

 Organizes the forcing data brought by the MainGenerator.
The user should be aware of the expected format of the forcing data for their model.
By default, this module creates forcing data in a folder names 'Forcing'. Such forcing data is
always created as timeseries where different columns hold different variables, ie:
(Precipitation_t0 | Temerature_t0 | Wind_t0, etc)
(Precipitation_t1 | Temerature_t1 | Wind_t1, etc)
(Precipitation_t2 | Temerature_t2 | Wind_t2, etc)
(Precipitation_t3 | Temerature_t3 | Wind_t3, etc)
and the data is divided in different files, were each one represents a single cell of the forcing inputs.
So, a 2 by 2 (2x2) model will generate 4 files, each one with the full timeseries of the forcing data.

[INPUTS]
WD_Path: This is the working folder provided by the MainGenerator module.
DataSet_Class: This is the dataset information provided by the MainGenerator module.
Forcing_Folder_Name: [Optional] Name of the subfolder where the forcing info will be stored. [Default = Forcing]
\ Forcing_File_Prefix: [Optional] Prefix for the name of the forcing files. [Default = data].
Date_Label_Format: [Optional] Format taken for the dates of the first column. (i.e 05/16/1997-05:30:00 can be expressed as %m/%d/%YYYY-%H:%M:%S )[Default = None] The files will not have dates.
For more information about python date formats go to [https://strftime.org/]
Subrange: [Optional] Subset of the dataset by giving the boundary limitation including x_min, x_max, y_min, y_max.
Mask_File: [Optional] Mask file for generating specific forcing data files, there are two types of mask files, which are inclusive and exclusive.\ In the mask file, it records the latitude and longitude of the cell each line, and for inclusive mask file, the ForcingDataFileGenerator module will only \ yield the forcing data files of the certain cell given in mask file. and in reverse, the exclusive mask file provided to create the forcing data files of the cells which is excluded in the mask file.\ The format of the mask file, should be [latitude],[longitude] each line, and user can make comment freely in any other line.

[OUTPUTS]
Ready: (True/False) starting flag for RunModuleAgent.
